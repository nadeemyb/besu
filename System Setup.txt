Tools and Applications

Development Tools
•	Remix IDE (Online) : for Solidity contract development and testing (Remix)
•	Visual Studio Code (VS Code): for scripting and API integration
•	Ubuntu Server (22.04 or 24.04 LTS): for blockchain node deployment
•	OpenJDK (Java 17 or 21): required for Besu execution
•	Node.js (v18+): supports DApp interaction and benchmarking tools
•	Python 3.10+ with pip: for backend scripting and Web3 integration
•	Web3.py: Python library for Ethereum node interaction
•	Web3.js: JavaScript library for DApp development
•       IPFS (Kubo): Distributed off-chain storage for encrypted and plaintext patient medical records.

Blockchain Components
•	Hyperledger Besu (v23.7+ / v25.8.0): Ethereum-compatible client for permissioned blockchain
•	Docker & Docker Compose: containerized multi-node deployment (validators, RPC, peers)
•	Genesis Block Configuration: defines initial state, accounts, and consensus protocol parameters
•	Consensus Algorithms Implemented: QBFT, IBFT2, and Clique (evaluated individually)
•	Quorum Explorer: used for node and transaction monitoring
•  InterPlanetary File System (IPFS): Stores patient medical records off-chain while blockchain stores only the corresponding CID and cryptographic hash.

Testing and Benchmarking Tools
•	Testing
•	Web3.py / Python automation scripts for patient registration, policy creation, consent management, IPFS interaction, blockchain transactions, and benchmarking.
•	Curl for JSON-RPC testing and validation
•	Benchmarking
•	Python-based performance scripts for measuring latency, throughput, gas usage, CPU, and memory utilization

Implementation Workflow
a. Configure Ubuntu Server and install required dependencies.
b. Install Docker, Docker Compose, OpenJDK, Python, and Hyperledger Besu.
c. Install and initialize the local IPFS daemon.
d. Configure the permissioned blockchain network and consensus protocol (QBFT, IBFT2, or Clique).
e. Generate the genesis block and initialize validator nodes.
f. Deploy blockchain nodes using Docker Compose.
g. Deploy the PBAC smart contract through Remix IDE using MetaMask.
h. Update the deployed contract address and ABI in the Python configuration.
i. Import patient datasets and consent information.
j. Upload patient medical records to IPFS and generate the corresponding CIDs and cryptographic hashes.
k. Register patient metadata and IPFS references on the blockchain.
l. Execute Normal, Delegate, and Emergency authorization scenarios.
m. Evaluate access requests and retrieve authorized records from IPFS after blockchain verification.
n. Monitor blockchain transactions and system performance using Quorum Explorer and the benchmarking scripts.
